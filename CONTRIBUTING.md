# Contributing to Seer

First off, thanks for helping us make agents more reliable. We want to keep this project high-quality and technically sound.

## The Legal Part (Keep it simple)

We use the **MIT** license. This ensures the core engine remains open for everyone.

We also offer commercial licenses to companies needing enterprise features, so we require all contributors to sign a **Contributor License Agreement (CLA)**. 

### How to sign:
1. Open a Pull Request.
2. Our **CLA Assistant** bot will comment on your PR.
3. Click the link in the comment to sign (it takes 30 seconds via GitHub OAuth).
4. Once signed, your PR will be automatically cleared for review.

## Technical Guidelines
- **Small PRs:** We prefer incremental changes over 2,000-line refactors.
- **Tests:** If you're fixing a bug in an agent node, include the `seer test` or eval run result.
- **Docs:** Update the relevant README if you change a tool's signature or a node's state schema.

## Questions?
If you're not sure if a feature fits the roadmap, open a **Discussion** or an Issue before writing the code.
